## 4. 数据库增量 SQL（建议顺序）

> 适用前提：**已有生产库数据**。生产环境请勿执行 `00_full_schema.sql`（内含建表/可能带删表逻辑风险）。

在已备份前提下，按“表结构 -> RBAC 导航/能力”顺序补跑以下脚本（均为幂等 `CREATE TABLE IF NOT EXISTS` / `INSERT ... ON DUPLICATE KEY UPDATE` 设计）：

| 建议顺序 | 文件 | 说明 |
|----------|------|------|
| 1 | `scripts/sql/run_26_add_semi_material_table.sql` | 创建 `semi_material` 主数据表（无外键约束） |
| 2 | `scripts/sql/run_27_semi_material_rbac.sql` | 写入半成品/物料导航与能力键；并给 `warehouse` 角色分配 `role_allowed_nav=semi_material` |
| 3 | `scripts/sql/run_28_add_bom_tables.sql` | 创建 `bom_header` / `bom_line`（无外键约束） |
| 4 | `scripts/sql/run_29_bom_rbac.sql` | 写入 BOM 导航与能力键；并给 `warehouse` 角色分配 `role_allowed_nav=bom` |
| 5 | `scripts/sql/run_30_add_production_tables.sql` | 创建预生产计划/工作单/缺料明细四张表（无外键约束） |
| 6 | `scripts/sql/run_31_production_rbac.sql` | 写入生产管理导航与能力键；并给 `warehouse` 角色分配 `role_allowed_nav=production` |

执行后可检查：

```sql
SHOW TABLES LIKE 'semi_material';
SHOW TABLES LIKE 'bom_header';
SHOW TABLES LIKE 'production_preplan';

-- 可选：确认导航/能力键是否已落库
SELECT code FROM sys_nav_item WHERE code IN ('semi_material','bom','production');
SELECT code FROM sys_capability WHERE code LIKE 'semi_material.%' OR code LIKE 'bom.%' OR code LIKE 'production.%';
```

## 5. 应用与发布步骤

1. 将**整份当前代码**同步到服务器（包含新增未跟踪文件）。
2. 在服务器 Python 虚拟环境里：`pip install -r requirements.txt`（若依赖有变才必做）。
3. 对照 `.env.example` 更新生产 `.env`：至少保证 `DATABASE_URL`、`SECRET_KEY` 正确。
4. 按第 4 节补跑 SQL：`run_26` -> `run_31`（如服务器已按完整建表脚本初始化，可酌情跳过表结构补丁）。
5. 重启 WSGI 进程（如 gunicorn / uwsgi / systemd）。
6. 登录后台：确认 `warehouse` 角色能看到新菜单（半成品/物料、BOM、生产管理）。
