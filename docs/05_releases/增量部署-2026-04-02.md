## 3. 通过 OpenClaw 部署（推荐流程）

OpenClaw 在此场景下用于**在目标机或受控会话中按步骤执行**：备份 → 执行 SQL → 同步代码 → 重启 → 验收。以下为建议指令级顺序（可按你方 OpenClaw 技能拆成多步）。

### 3.1 部署前

- 全库备份 `sydixon_order`（至少含 RBAC、业务表、编排相关新表）。
- 确认部署分支与本次文档一致；记录当前 Git 提交哈希，便于回滚。
- 确认应用环境变量与数据库连接可用（含 `DATABASE_URL` / `utf8mb4`）。

### 3.2 执行 SQL（OpenClaw 逐步执行）

- 按 **第 1 节表格顺序** 执行 `mysql` 或等价工具，**每步成功后再执行下一步**。
- 若脚本含导航/能力变更，执行后需 **重启应用进程** 或按项目既有方式刷新 RBAC 缓存。

### 3.3 发布代码并重启

- 将仓库同步到目标机（与本次变更一致）。
- 安装/校验 Python 依赖：`pip install -r requirements.txt`（在部署虚拟环境中）。
- 重启 Web / Worker 进程（确保菜单、能力、蓝图注册全部生效）。

### 3.4 OpenClaw 侧（若使用对话 API / 技能）

- 应用侧 OpenClaw 全局 Key / 用户令牌能力见 `项目整体部署.md` 与 `docs/openclaw-skill/README.md`。
- 若本次更新了 `docs/openclaw-skill/SKILL.md`，部署后需将技能文件 **复制到 OpenClaw 技能目录**（README 中 `cp` 命令），避免客户端仍用旧说明。

### 3.5 最小验收

- 库存：三个录入入口可访问且权限与菜单一致。
- Orchestrator：引擎相关页面或 API（若已开放）可按 `docs/orchestrator_uat_checklist.md` 抽样。
- 机台/人员：排班模板、预约列表与表单可打开且无 500。
- HR 能力：员工能力、部门能力映射页面可访问。
- 生产：与预算/成本相关入口与页面与库表一致。
