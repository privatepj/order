# 增量部署说明（2026-04-11）

本文档对应 **截至 2026-04-11** 工作区中尚未提交的变更，面向 **生产/测试库 `sydixon_order` 的增量上线**，并便于通过 **OpenClaw** 在服务器上按步骤执行（备份 → SQL 按序 → 同步代码 → 安装依赖 → 重启 → 验收）。

**与上一版归档的关系**：若服务器已按 `docs/05_releases/增量部署-2026-04-02.md` 完成 `run_42`～`run_60`，本次只需继续执行下文 **第 1 节** 所列 **`run_61`～`run_76`**（已执行的脚本跳过，但需有对账记录）。

## 0. 红线与前置

- **`scripts/sql/run_*.sql` 只增不改**：已发布过的历史文件名与语义不得就地改写（见 `.cursor/rules/incremental-sql-append-only.mdc`）。
- **禁止数据库级 FOREIGN KEY**（见 `.cursor/rules/no-database-foreign-keys.mdc`）。
- **生产勿用全量重建**：`scripts/sql/00_full_schema.sql` 仅作新环境/对齐参考；存量库请只跑本节列出的增量 `run_*.sql`。
- **部署前**：全库备份；记录当前应用 Git 提交哈希与已执行过的 `run_*` 列表，便于回滚与对账。
- **环境**：确认 `DATABASE_URL`、`utf8mb4`、虚拟环境与线上一致；`pip install -r requirements.txt` 在部署窗口内执行。

## 1. 增量 SQL 执行顺序（必须按序；失败即停）

数据库：`sydixon_order`

| 顺序 | 脚本 | 概要 |
|------|------|------|
| 1 | `scripts/sql/run_61_piece_rate.sql` | 工种计件单价表 |
| 2 | `scripts/sql/run_62_dept_piece_rate_rbac.sql` | 工种计件单价：导航 + 能力 + 财务角色菜单 |
| 3 | `scripts/sql/run_63_hr_work_type_refactor.sql` | HR 工种解耦（部门/工种/员工绑定、兼容回填、导航与能力） |
| 4 | `scripts/sql/run_64_procurement_supplier_batch_and_compare.sql` | 供应商主数据、供应关系、请购/采购/对账等配套表与导航能力 |
| 5 | `scripts/sql/run_65_procurement_default_supplier_cleanup.sql` | 默认供应商数据清理（停用、重复默认等） |
| 6 | `scripts/sql/run_66_procurement_material_management.sql` | 物料管理相关约束/清理（与采购默认供应商逻辑衔接） |
| 7 | `scripts/sql/run_67_delivery_method.sql` | 送货单 `delivery_method`（快递/自配送/自提），并按历史快递字段回填 |
| 8 | `scripts/sql/run_67_rbac_mixed_mode_backfill.sql` | **与同号另一文件并列**：混合态 RBAC 从 `allowed_menu_keys` 向 `role_allowed_nav` 补齐（可重复执行） |
| 9 | `scripts/sql/run_68_order_item_is_spare.sql` | 订单明细 `is_spare`（备品） |
| 10 | `scripts/sql/run_69_move_schedule_menus_to_production.sql` | 机台排班、人员排班菜单挂到「生产」导航下 |
| 11 | `scripts/sql/run_70_production_routing_target_kind.sql` | 产品路由支持成品/半成品统一对象（`target_kind` / `target_id`） |
| 12 | `scripts/sql/run_71_production_operation_plan_commit.sql` | 工序计划确认字段 + `production_schedule_commit_row` 等 |
| 13 | `scripts/sql/run_72_inventory_reservation.sql` | 库存预留表 `inventory_reservation`（预计划占用，非出库流水） |
| 14 | `scripts/sql/run_76_decimal_scale_8.sql` | 多表业务数量/金额列统一 `decimal(26,8)`（**含对 `inventory_reservation.reserved_qty` 的 MODIFY**，须在 `run_72` 之后） |

**特别说明**

- 存在 **两个 `run_67_*.sql`**（历史编号并列）：请按上表 **先 `run_67_delivery_method.sql`，再 `run_67_rbac_mixed_mode_backfill.sql`**。二者无强 schema 依赖，但固定顺序便于运维对账。
- **`run_76` 必须排在 `run_72` 之后**：`run_76` 会 `ALTER` 已存在的 `inventory_reservation` 列精度；若全新库只跑全量 schema 可忽略此顺序，增量环境请遵守本表。
- 仓库中 **`scripts/sql/user_today/*.sql`** 为本地全链路种子/校验脚本，**默认不在生产增量流程中执行**；除非明确需要造数环境。

## 2. 业务变更摘要（按域）

便于 OpenClaw 或人工验收时对照。

| 域 | 要点 |
|----|------|
| **HR** | 工种计件单价页面与权限；工种与部门解耦（主/辅工种、兼容回填）；相关模板与路由、`hr_work_type_svc` 等。 |
| **采购** | 供应商/物料维护、请购打印、入库对账、采购单 Excel 模板路径；默认供应商与数据清理逻辑。 |
| **送货** | 配送方式枚举与展示/导出；`delivery_method` 工具与测试 `tests/test_delivery_method.py`。 |
| **订单** | 明细备品标记 `is_spare` 与界面/服务联动。 |
| **生产** | 路由目标成品/半成品；预计划手工排程与计划确认（commit 行表）；排班菜单归属生产；生产排程/测算与 BOM 门闸等调整（见 `production_*` 路由与服务）。 |
| **库存** | 预计划测算按 ATP 占用并写入预留；预留释放与删除预计划联动；`inventory_reservation` 模型与服务。 |
| **数值精度** | 多模型与 `decimal_scale` 工具对齐 `decimal(26,8)`，与 `run_76` 一致。 |
| **RBAC / 菜单** | 能力键与菜单同步多处路由；混合态角色菜单回填脚本。 |
| **OpenClaw** | `app/openclaw/routes.py` 等若有变更，部署后注意令牌与能力是否需重新下发或缓存刷新。 |

更细的变更说明可参考工作区内 `docs/changes/` 下条目（如 `2026-04-11_inventory_reservation_atp.md`、`2026-04-11_production_preplan_manual_schedule.md`、`2026-04-11_decimal_scale_8.md` 等）。

## 4. 通过 OpenClaw 执行部署（建议指令顺序）

将下列拆成 OpenClaw 多步任务时，保持顺序即可。

1. **备份** `sydixon_order`（物理备份或从库快照，按你方规范）。
2. **执行 SQL**：按 **第 1 节表格** 从 `run_61` 到 `run_76` 逐条执行；每条成功后再执行下一条；已执行过的跳过并记录。
3. **同步代码**：`git fetch` / `git checkout <分支或标签>`，确保与通过验收的提交一致。
4. **依赖**：在运行应用的 venv 中执行 `pip install -r requirements.txt`。
5. **重启**：重启 gunicorn/uwsgi/systemd 等应用进程（使菜单、能力、蓝图注册生效）。
6. **RBAC 缓存**：若生产环境有菜单/能力缓存，按既有方式 **刷新或短暂清空缓存**（脚本与代码已改导航/能力时尤其重要）。
7. **OpenClaw 技能文件**（若使用客户端技能）：若本次包含 `docs/openclaw-skill/SKILL.md` 或 `docs/04_ai/` 下技能变更，按项目 README 将技能同步到 OpenClaw 技能目录，避免客户端仍引用旧说明。

## 5. 最小验收清单（建议）

- **HR**：工种列表/表单、工种计件单价列表/表单可打开；财务角色能看到计件单价菜单（与 `run_62` 一致）。
- **采购**：供应商/物料列表与表单、请购打印、入库对账入口可访问；默认供应商行为符合预期。
- **送货**：新建/编辑送货单可选配送方式；列表与打印展示正确。
- **订单**：订单明细可标记备品且保存后回显正确。
- **生产**：产品路由支持选半成品（若已配置）；预计划详情中手工排程/计划确认相关操作无 500；生产主导航下可见机台/人员排班菜单。
- **库存**：预计划测算后产生预留、删除/释放预计划后预留清理（结合 `docs/changes/2026-04-11_inventory_reservation_atp.md` 抽样）。
- **数值**：关键数量/单价保存后小数位与前端展示符合 8 位小数策略（与 `run_76` 一致）。
