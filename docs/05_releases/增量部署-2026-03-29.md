## 5. 应用与 OpenClaw 发布步骤

1. 将**整份**当前代码同步到服务器（含上表「未跟踪」文件）。
2. `pip install -r requirements.txt`（依赖有变时）。
3. 对照 **`.env.example`** 更新生产 `.env`：  
   - 必填：`DATABASE_URL`、`SECRET_KEY`  
   - 库存/送货：`INVENTORY_DEFAULT_STORAGE_AREA`（若使用自动出库）  
   - OpenClaw：`OPENCLAW_API_KEY`（可选全局 Key）、`OPENCLAW_RATE_LIMIT_PER_MINUTE`（可选）
4. 按第 4 节补跑 SQL；在 **RBAC** 中为需要使用 OpenClaw 的用户角色勾选新增 `openclaw.*` 能力（种子脚本写入 `sys_capability`，角色授权需在后台配置）。
5. 用户令牌：在应用虚拟环境中执行 `flask openclaw-token-create <用户名>`（可选 `--days`、`--label`）；撤销用 `flask openclaw-token-revoke <token_id>`。CLI 会提示将明文令牌配置到 OpenClaw 侧（如环境变量 `SYDIXON_ORDER_API_KEY`）。另可选 `scripts/admin_openclaw_token.py`（按脚本说明使用）。
6. 重启 WSGI 进程（gunicorn / uwsgi / systemd 等）。
7. OpenClaw 技能文档：`docs/openclaw-skill/SKILL.md`、`README.md` 已随本次变更更新，部署后让 Agent 读取最新技能说明。
