# 工厂订单系统 - 整体部署说明

本文档供使用 openclaw 或手工部署时参考，按顺序执行即可完成环境与运行配置。

---

## 1. 环境要求

- **Python**：3.8 或以上
- **MySQL**：5.7 或 8.0
- 操作系统：Windows / Linux 均可

---

## 2. 克隆与依赖

（若为 git 仓库）

```bash
git clone <仓库地址>
cd sydixon-order
```

安装 Python 依赖：

```bash
pip install -r requirements.txt
```

---

## 3. 数据库

### 3.1 创建数据库

在 MySQL 中执行：

```sql
CREATE DATABASE IF NOT EXISTS sydixon_order DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 3.2 执行建表脚本

**推荐**使用整合后的 SQL 目录（新库只需执行一个文件）：

```bash
mysql -u 用户名 -p sydixon_order < scripts/sql/00_full_schema.sql
```

或在 MySQL 客户端中：

```sql
USE sydixon_order;
SOURCE /path/to/scripts/sql/00_full_schema.sql;
```

执行后将创建所有业务表及初始角色、管理员账号。

若从旧版本库升级，请执行 `scripts/sql/01_migrations_for_old_db.sql`，详见 `scripts/sql/README.md`。

---

## 4. 环境变量

复制示例配置并填写实际值：

```bash
copy .env.example .env
```

编辑 `.env`，必填项：

| 变量 | 说明 | 示例 |
|------|------|------|
| `SECRET_KEY` | Flask 会话密钥，生产环境务必改为随机字符串 | `your-secret-key-change-in-production` |
| `DATABASE_URL` | MySQL 连接串 | `mysql+pymysql://user:password@localhost:3306/sydixon_order` |
| `FLASK_ENV` | 开发用 development，生产用 production | `production` |
| `OPENCLAW_API_KEY` | （可选）OpenClaw 全局 Key，**不经过 RBAC**；可与用户令牌并存。不配则须使用 `flask openclaw-token-create` 签发的令牌 | 随机字符串 |
| `OPENCLAW_RATE_LIMIT_PER_MINUTE` | （可选）OpenClaw 每 IP 每分钟上限，`0` 关闭 | `120` |

`DATABASE_URL` 格式：`mysql+pymysql://用户名:密码@主机:端口/数据库名`

---

## 5. 运行应用

### 开发环境

```bash
python run.py
```

默认监听 `http://0.0.0.0:5000`，浏览器访问即可。

### 生产环境（推荐）

使用 gunicorn（需先安装：`pip install gunicorn`）：

```bash
gunicorn -w 4 -b 0.0.0.0:5000 "app:create_app()"
```

- `-w 4`：4 个 worker，可按 CPU 核数调整  
- `-b 0.0.0.0:5000`：监听所有网卡的 5000 端口  

若需指定配置文件或环境变量，可先导出再执行：

```bash
set SECRET_KEY=xxx
set DATABASE_URL=mysql+pymysql://...
gunicorn -w 4 -b 0.0.0.0:5000 "app:create_app()"
```

## 6. 可选：Nginx 反向代理与 HTTPS

在服务器前加一层 Nginx，示例配置（仅 HTTP）：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

如需 HTTPS，可配合 Let's Encrypt 等配置 `ssl_certificate`。

---

## 7. 可选：进程守护（Linux）

使用 systemd 保持应用常驻（示例单元文件 `/etc/systemd/system/sydixon-order.service`）：

```ini
[Unit]
Description=Factory Order System
After=network.target mysql.service

[Service]
Type=notify
User=www-data
WorkingDirectory=/path/to/sydixon-order
Environment="PATH=/path/to/venv/bin"
EnvironmentFile=/path/to/sydixon-order/.env
ExecStart=/path/to/venv/bin/gunicorn -w 4 -b 127.0.0.1:5000 "app:create_app()"
Restart=always

[Install]
WantedBy=multi-user.target
```

然后：

```bash
sudo systemctl daemon-reload
sudo systemctl enable sydixon-order
sudo systemctl start sydixon-order
```

## 9. 部署检查清单

- [ ] Python 3.8+ 已安装  
- [ ] MySQL 已安装并已创建数据库 `sydixon_order`  
- [ ] 已执行 `scripts/sql/00_full_schema.sql`  
- [ ] 已配置 `.env`（`SECRET_KEY`、`DATABASE_URL`）  
- [ ] 已执行 `pip install -r requirements.txt`  
- [ ] 开发：已执行 `python run.py`；生产：已用 gunicorn 或 systemd 启动  
- [ ] 防火墙或安全组已放行 5000 端口（或 Nginx 对应端口）  
- [ ] 首次登录后已修改 admin 密码  
- [ ] （可选）若使用 OpenClaw 对话：已配置全局 `OPENCLAW_API_KEY` 和/或执行 `run_23`+`run_24` SQL 并用 `flask openclaw-token-create <用户名>` 签发用户令牌；角色勾选 OpenClaw 相关细项（含预览与确认制所需能力）；已安装技能至 `~/.openclaw/skills/sydixon-order/`

## 10. 可选：OpenClaw 对话接入

与 OpenClaw（龙虾）对话即可创建订单、送货单（如「给 XX 客户送 50K A产品」）。

### 10.1 订单系统侧

1. **鉴权（二选一或并存）**  
   - **全局 Key**：在 `.env` 中配置 `OPENCLAW_API_KEY`（与技能环境变量 `SYDIXON_ORDER_API_KEY` 可填同一值）。该 Key **不受 RBAC 限制**，等同高权限调用。  
   - **用户令牌（推荐）**：在服务器上执行 `flask openclaw-token-create <登录名>`（需已激活用户且非 pending），将输出的 `oc_…` 令牌配入 `SYDIXON_ORDER_API_KEY`。数据库需有表 `user_api_token` 及 OpenClaw 能力定义（新库已含于 `00_full_schema.sql`；旧库依次执行 `run_23_openclaw_user_token_and_caps.sql`、`run_24_openclaw_confirm_flow_caps.sql`）。在 **角色管理** 中勾选 **OpenClaw** 下所需细项（含读接口、预览、写接口，如 `openclaw.order.preview` 与 `openclaw.order.create` 等）。  
2. 可选环境变量 `OPENCLAW_RATE_LIMIT_PER_MINUTE`（默认每分钟每 IP 120 次，`0` 关闭）。  
3. 确保默认快递「顺丰」已在系统中维护（快递公司 + 运单号池），否则 API 创建送货单时需显式传 `express_company_id`。  
4. 专用接口前缀：`/api/openclaw`，请求头 `X-API-Key` 或 `Authorization: Bearer <凭据>`。  
5. **编号规则**：订单号、送货单号、运单占号均由 **服务端生成或分配**，客户端/技能 **不得** 自拟单号写入请求体。  
6. **确认制技能**：对话侧应按 `docs/openclaw-skill/SKILL.md` 先预览（`orders/preview`、`deliveries/preview`）再创建；新建客户/客户产品亦有对应 POST。

### 10.2 接口一览

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/openclaw/companies` | 经营主体列表（选 `company_id`） |
| GET | `/api/openclaw/products?q=` | 系统产品搜索 |
| GET | `/api/openclaw/customers?q=` | 客户搜索（**q 必填**，至少 2 字符；仅 id、简称；响应不含联系人/手机号） |
| POST | `/api/openclaw/customers` | 新建客户（单主体，JSON） |
| GET | `/api/openclaw/customer-products?customer_id=&q=` | 客户产品列表 |
| POST | `/api/openclaw/customer-products` | 新建客户产品绑定 |
| GET | `/api/openclaw/deliveries/pending-items?customer_id=&order_id=` | 待发货订单行 |
| POST | `/api/openclaw/orders/preview` | 订单创建校验与摘要（不写库） |
| POST | `/api/openclaw/orders` | 创建订单 |
| POST | `/api/openclaw/deliveries/preview` | 送货单创建校验与摘要（不写库） |
| POST | `/api/openclaw/deliveries` | 创建送货单（可选 `order_id`） |

### 10.3 OpenClaw 技能安装

将项目内技能文件复制到 OpenClaw 技能目录：

```bash
mkdir -p ~/.openclaw/skills/sydixon-order
cp docs/openclaw-skill/SKILL.md ~/.openclaw/skills/sydixon-order/
```

在 OpenClaw 运行环境中配置：

- `SYDIXON_ORDER_URL`：订单系统 base URL（如 `http://服务器:5000`）。
- `SYDIXON_ORDER_API_KEY`：填全局 `OPENCLAW_API_KEY`，或 `flask openclaw-token-create` 生成的用户令牌（`oc_` 开头）。

详见 `docs/openclaw-skill/README.md`。
