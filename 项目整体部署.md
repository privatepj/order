# 工厂订单系统 - 整体部署说明

本文档供使用 openclaw 或手工部署时参考，按顺序执行即可完成环境与运行配置。

---

## 1. 环境要求

- **Python**：3.8 或以上
- **MySQL**：5.7 或 8.0
- 操作系统：Windows / Linux 均可

---

## 2. 克隆与依赖

（若为 git 仓库）

```bash
git clone <仓库地址>
cd sydixon-order
```

安装 Python 依赖：

```bash
pip install -r requirements.txt
```

---

## 3. 数据库

### 3.1 创建数据库

在 MySQL 中执行：

```sql
CREATE DATABASE IF NOT EXISTS sydixon_order DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 3.2 执行建表脚本

**推荐**使用整合后的 SQL 目录（新库只需执行一个文件）：

```bash
mysql -u 用户名 -p sydixon_order < scripts/sql/00_full_schema.sql
```

或在 MySQL 客户端中：

```sql
USE sydixon_order;
SOURCE /path/to/scripts/sql/00_full_schema.sql;
```

也可使用根目录的 `scripts/init_db.sql`（与上述内容一致）。执行后将创建所有业务表及初始角色、管理员账号。

若从旧版本库升级，请执行 `scripts/sql/01_migrations_for_old_db.sql`，详见 `scripts/sql/README.md`。

---

## 4. 环境变量

复制示例配置并填写实际值：

```bash
copy .env.example .env
```

编辑 `.env`，必填项：

| 变量 | 说明 | 示例 |
|------|------|------|
| `SECRET_KEY` | Flask 会话密钥，生产环境务必改为随机字符串 | `your-secret-key-change-in-production` |
| `DATABASE_URL` | MySQL 连接串 | `mysql+pymysql://user:password@localhost:3306/sydixon_order` |
| `FLASK_ENV` | 开发用 development，生产用 production | `production` |
| `OPENCLAW_API_KEY` | （可选）OpenClaw 对话接入用 API Key；不配置则 `/api/openclaw` 接口返回 503 | 随机字符串，与 OpenClaw 技能中配置一致 |

`DATABASE_URL` 格式：`mysql+pymysql://用户名:密码@主机:端口/数据库名`

---

## 5. 运行应用

### 开发环境

```bash
python run.py
```

默认监听 `http://0.0.0.0:5000`，浏览器访问即可。

### 生产环境（推荐）

使用 gunicorn（需先安装：`pip install gunicorn`）：

```bash
gunicorn -w 4 -b 0.0.0.0:5000 "app:create_app()"
```

- `-w 4`：4 个 worker，可按 CPU 核数调整  
- `-b 0.0.0.0:5000`：监听所有网卡的 5000 端口  

若需指定配置文件或环境变量，可先导出再执行：

```bash
set SECRET_KEY=xxx
set DATABASE_URL=mysql+pymysql://...
gunicorn -w 4 -b 0.0.0.0:5000 "app:create_app()"
```

## 6. 可选：Nginx 反向代理与 HTTPS

在服务器前加一层 Nginx，示例配置（仅 HTTP）：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

如需 HTTPS，可配合 Let's Encrypt 等配置 `ssl_certificate`。

---

## 7. 可选：进程守护（Linux）

使用 systemd 保持应用常驻（示例单元文件 `/etc/systemd/system/sydixon-order.service`）：

```ini
[Unit]
Description=Factory Order System
After=network.target mysql.service

[Service]
Type=notify
User=www-data
WorkingDirectory=/path/to/sydixon-order
Environment="PATH=/path/to/venv/bin"
EnvironmentFile=/path/to/sydixon-order/.env
ExecStart=/path/to/venv/bin/gunicorn -w 4 -b 127.0.0.1:5000 "app:create_app()"
Restart=always

[Install]
WantedBy=multi-user.target
```

然后：

```bash
sudo systemctl daemon-reload
sudo systemctl enable sydixon-order
sudo systemctl start sydixon-order
```

## 9. 部署检查清单

- [ ] Python 3.8+ 已安装  
- [ ] MySQL 已安装并已创建数据库 `sydixon_order`  
- [ ] 已执行 `scripts/sql/00_full_schema.sql`（或 `scripts/init_db.sql`）  
- [ ] 已配置 `.env`（`SECRET_KEY`、`DATABASE_URL`）  
- [ ] 已执行 `pip install -r requirements.txt`  
- [ ] 开发：已执行 `python run.py`；生产：已用 gunicorn 或 systemd 启动  
- [ ] 防火墙或安全组已放行 5000 端口（或 Nginx 对应端口）  
- [ ] 首次登录后已修改 admin 密码  
- [ ] （可选）若使用 OpenClaw 对话：已配置 `OPENCLAW_API_KEY`，并已安装技能至 `~/.openclaw/skills/sydixon-order/`  

完成以上步骤后，即可使用 openclaw 或浏览器访问系统进行订单、送货与对账导出。

## 10. 可选：OpenClaw 对话接入

与 OpenClaw（龙虾）对话即可创建订单、送货单（如「给 XX 客户送 50K A产品」）。

### 10.1 订单系统侧

1. 在 `.env` 中配置 `OPENCLAW_API_KEY`（与 OpenClaw 技能环境变量 `SYDIXON_ORDER_API_KEY` 一致）。
2. 确保默认快递「顺丰」已在系统中维护（快递公司 + 运单号池），否则 API 创建送货单时需显式传 `express_company_id`。
3. 专用接口前缀：`/api/openclaw`，鉴权方式：请求头 `X-API-Key` 或 `Authorization: Bearer <key>`。

### 10.2 接口一览

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/openclaw/customers?q=` | 客户列表（仅 id、简称，不暴露完整信息） |
| GET | `/api/openclaw/customer-products?customer_id=&q=` | 客户产品列表 |
| GET | `/api/openclaw/deliveries/pending-items?customer_id=&order_id=` | 待发货订单行（order_id 可选） |
| POST | `/api/openclaw/orders` | 创建订单（JSON body） |
| POST | `/api/openclaw/deliveries` | 创建送货单（JSON body，默认今天+顺丰） |

### 10.3 OpenClaw 技能安装

将项目内技能文件复制到 OpenClaw 技能目录：

```bash
mkdir -p ~/.openclaw/skills/sydixon-order
cp docs/openclaw-skill/SKILL.md ~/.openclaw/skills/sydixon-order/
```

在 OpenClaw 运行环境中配置：

- `SYDIXON_ORDER_URL`：订单系统 base URL（如 `http://服务器:5000`）。
- `SYDIXON_ORDER_API_KEY`：与订单系统 `.env` 中 `OPENCLAW_API_KEY` 相同。

详见 `docs/openclaw-skill/README.md`。
